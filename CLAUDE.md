# 测试规则（强制）

**禁止视觉测试。所有验证必须用代码完成。**

- 不允许使用截图、computer-use、屏幕识别等视觉手段来验证程序行为。
- 任何东西，只要能用代码测试，就必须用代码测试。例如：
  - Rust 后端逻辑 → `cargo test` 单元测试 / 集成测试
  - Tauri command → 直接调用函数写测试，或通过脚本驱动
  - 前端逻辑 → 用脚本/无头方式断言 DOM 和行为，不靠肉眼看截图
  - 进程是否启动 → 检查进程、退出码、日志输出
- 验证标准是程序化断言（exit code、stdout、日志、测试结果），不是"看起来对"。
